## Resume Mode

### プロジェクト概要

複数の日本の ASP（A8.net、Access Trade、afb）を横断して、同一キャンペーンの成果を比較できるアフィリエイト成果比較プラットフォーム。2 人のスタートアップ環境で事業オーナーとしてビジネスと技術開発を主導。

### 課題

個人アフィリエイターは同じ案件を複数 ASP に出稿できるが、成果や条件は ASP ごとに大きく異なる。成果データを集約・比較する集中ツールがなく、手作業で各プラットフォームを確認するしかなかった。

### 役割と責任

- **事業オーナー / プロジェクトリード:** オーナーと 2 名体制で企画から実行までを統括
- **技術実装:** RPA ツール（irobo、現 Autoro）と JavaScript を用いてスクレイピングを単独実装
- **自律学習:** 実装と並行してオンライン講座で JavaScript 基礎を習得
- **ビジネス戦略:** 親会社の広告代理店リレーションを活用したマネタイズモデルを設計

### 技術実行

- 主要 3 ASP からの自動データ抽出を RPA ベースのスクレイピングで実装
- 取得データのパースと分析ロジックを JavaScript で構築
- 複数 ASP ダッシュボードにおける認証フローと UI 差分を個別対応
- 同一キャンペーン比較を提示するダッシュボード構想を設計

### 課題と学び

**技術的課題:**

- 共通 ID がないためキャンペーンの突合・重複排除が困難
- 技術基盤の乏しさを埋めるため高速でスキル習得が必要
- RPA ツールの制約と要件のバランス取り

**主な学び:**

- JavaScript の基礎とスクレイピング手法
- コンピュータの処理速度と自動化が手作業を置き換える可能性
- アイデア創出から PoC 検証までのビジネス開発ライフサイクル

### 成果・インパクト

- コードが事業価値と収益を生み得るという基礎理解を獲得
- ソフトウェアがアイデアをスケールさせるレバレッジであると認識
- **キャリアインパクト:** ビジネスとプログラミングへのキャリア転換を促し、その後の起業・技術活動の土台を形成

### 技術スタック

- **RPA:** irobo（現 Autoro）
- **言語:** JavaScript（初めて触れた言語）
- **プラットフォーム:** A8.net、Access Trade、afb（日本の ASP）

### 中止理由

中止の理由:

1. プラットフォーム間でのキャンペーン突合の技術的複雑性
2. 製品要件を満たす技術力の不足
3. 追加投資前に PoC 検証が必要だったため

### 振り返りと改善案

もし今取り組むなら:

- 需要検証のための超短期 PoC を先に作る
- テストマーケティングで PMF を検証する
- スクレイピング依存ではなく ASP と直接提携する
- 近代的なスクレイピングフレームワークで堅牢に抽出する

## Storytelling Mode

### はじまり — 学生が偶然ビジネスに踏み込む

2019 年。ビジネスを「触れられるもの」と考えたこともなかった学生だった。HTML/CSS は触れたことがあったが、JavaScript は未知。そのとき現れたのが ACE だった。

Introduction Inc. にインターンで参加したが、典型的なインターンとは違い、オフィスというよりガレージのような場所でオーナーと 2 人きり。場違いに大きな肩書「**Project Lead**」を与えられたが、その意味もまだ分かっていなかった。

### すべてを動かした課題

発想は単純だった。A8.net、Access Trade、afb を人力で毎回チェックし、同一案件のどこが一番条件が良いかを探す——その退屈で時間がかかる作業を自動化したい。データを取って、比べて、最適を示す。簡単そうに見えた。

だが簡単ではなかった。

### 初めて「動いた」瞬間

スクレイピングも JavaScript も初めて。オーナーに紹介された RPA ツール（irobo、現 Autoro）を使うことにし、戦略もなく、オンライン講座をかじりながら手を動かした。

ある日、スクリプトが走り、A8.net の数字が自動で画面に現れた。信じられないほど速く、人が何時間もかける作業が数秒で終わる。そのとき悟った——**機械は疲れない**。飽きない。ただ黙々とやり続け、瞬時に終わらせる。_これができるなら、他の「仕事」も置き換えられるのでは？_ と考えた。

### 思考の速さ、機械の速さ

JavaScript でパース処理を書きながら、その速さに驚き続けた。比較や整理に何時間もかかる作業がミリ秒で終わる。これは単なる興奮ではなく、**レバレッジ**だと気づいた。アイデアにコードを掛け合わせれば、人間の限界を超えて増幅できる。

### 壁

しかし現実は厳しかった。ASP ごとにデータ形式が違い、共通 ID もない。まだ未熟なスキルでは解けず、プロジェクトは止まった。

### 失ったものと得たもの

ACE はローンチに至らず、ダッシュボードもユーザーもできなかった。紙の上では失敗だ。

それでも、もっと大きなものを得た。

### 人生を変えた教訓

**ビジネスは遠い世界ではない**。アイデアを持つ人々が試し、失敗し、やり直すだけ。そして自分もその一人になれると知った。

資本主義についても学んだ。**機会はそこら中にあり、それを掴むかどうかは自分次第**だということ。

そして何より、**エンジニアリングはレバレッジ**だ。コードがあれば現実を変えられる。いつかではなく、**今**。

この 4 か月が、「価値あるものを作れる」「世界を変えられる」と信じるきっかけになった。学び、失敗し、続ける覚悟があれば。

### 芽生えた種

製品としての ACE は成功しなかったが、**触媒**として成功した。プログラミングとスタートアップへ進む火種となり、「想像できるなら作れる」というマインドを植え付けた。

その後のすべてのコード、ビジネスアイデア、リスクテイクは 2019 年のこの 4 か月に遡る。スクラップな RPA、JavaScript の入門、そしてこう気づいた瞬間——

**世界は可塑的であり、自分にはそれを形作る道具がある。**
